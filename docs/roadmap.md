## 🐛 已知问题

1. 直接打开 `index.html`（file:// 协议）时，PWA Service Worker 不可用（浏览器限制），但所有功能正常
2. AI 分析仅在配置了有效 API Key 时可用
3. 初次使用时，需在设置中保存配置后才会显示「已配置」状态
4. 时间记录不支持跨天（如 `2300-0100` 会算成负数）

## 💡 设计决策记录 (ADR)

### ADR-001: 单 HTML 文件架构
- **时间**: 2026-05-23
- **决策**: 所有功能（HTML/CSS/JS）放在一个 `index.html` 中
- **理由**: 零构建、零依赖，双击即可使用，降低使用门槛
- **权衡**: 代码维护性不如拆分文件，34KB 的大小在可接受范围

### ADR-002: IndexedDB 本地存储
- **时间**: 2026-05-23
- **决策**: 使用浏览器 IndexedDB 存储所有用户数据
- **理由**: 无需后端服务器，数据完全由用户掌控，离线可用
- **权衡**: 清除浏览器数据会丢失，需用户自行备份

### ADR-003: OpenAI 兼容 API
- **时间**: 2026-05-23
- **决策**: 使用 OpenAI Chat Completions API 格式接入多家 AI
- **理由**: DeepSeek / 千问 / 豆包 都兼容此格式，切换成本最低
- **权衡**: 各家模型能力有差异，部分高级功能（如 function calling）可能不通用
